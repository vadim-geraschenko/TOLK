# Background Video By Images Sequence Animation

Этот файл фиксирует актуальный технический подход к фоновой сцене `about.html`.

## Основной файл
- [about.html](/home/rotvein/Desktop/TOLK_design/docs/design/pages/about/about.html)

Именно `about.html` считать рабочим baseline.

## Базовый подход
Для страницы используется image sequence, а не прямой scroll-scrub по `mp4`.

Причина:
- для этого кейса sequence оказался стабильнее и предсказуемее;
- текстовая страница выигрывает от контролируемого fixed background;
- прямой `video.currentTime` scrub здесь не нужен.

## Актуальная конфигурация
- `60` кадров;
- формат: `webp`;
- отдельные наборы для `desktop` и `mobile`;
- `desktop`: около `900px`;
- `mobile`: около `600px`;
- preload-буфер `±6` кадров.

Ассеты:
- `docs/design/pages/about/assets/angel-sequence/desktop`
- `docs/design/pages/about/assets/angel-sequence/mobile`

## Desktop-поведение
На desktop активен scroll-driven sequence:
- scroll position переводится в `progress`;
- `progress` маппится в индекс кадра;
- кадр подставляется в fixed `<img id="story-frame">`;
- preload идёт только вокруг текущего кадра.

Дополнительно:
- сцена в начале темнее;
- по мере скролла становится чуть яснее;
- поверх остаётся лёгкая зернистость;
- облака на desktop включены как отдельные декоративные слои.

## Mobile-поведение
На mobile sequence отключается полностью.

Текущее правило:
- при ширине `760px` и меньше не запускать render-loop sequence;
- не гонять покадровую подмену;
- показывать один статичный кадр ангела;
- сохранять текущий оверлей и атмосферу сцены;
- облака на mobile не показывать.

Текущий статичный mobile-кадр:
- `./assets/angel-sequence/mobile/frame-028.webp`

## Инициализация
Для desktop:
- стартовый `src` у фонового изображения не задан в HTML;
- сначала вычисляется актуальная позиция скролла;
- только потом подставляется нужный кадр;
- фон становится видимым после загрузки первого корректного кадра.

Для mobile:
- при старте сразу подставляется статичный mobile-кадр;
- sequence-цикл не запускается.

## Облака
Текущее правило:
- desktop: облака допускаются как дополнительный слой;
- mobile: облака отключены полностью ради производительности.

Это важно:
- если страница на телефонах снова начинает лагать, сначала выключать декоративные слои;
- не пытаться любой ценой сохранить одинаковую сценографию между desktop и mobile.

## Что считать baseline
Baseline для текущей версии:
- desktop: image sequence + мягкий оверлей + зернистость + desktop-облака;
- mobile: один статичный кадр ангела + тот же общий оверлей, без облаков и без sequence-анимации.

## Чего не делать
- не возвращать direct `mp4` scrub как дефолт;
- не включать на mobile sequence и облака одновременно;
- не добавлять тяжёлые слои, если они ухудшают плавность страницы;
- не считать desktop-эффекты обязательными для мобильной версии.
